# TOJIMASAYA.COM 改善提案書

## 📊 サイト概要
- **サイト名**: Toji Masaya Portfolio
- **主要コンテンツ**: 写真家・ライターのポートフォリオサイト
- **主要ページ**: Home, About, Gallery, Writings, Gear, Trips, Hong Kong
- **技術スタック**: HTML/CSS/JavaScript（バニラ）、一部Tailwind CSS

---

## 🔍 現状分析

### ✅ 良い点
1. **明確なコンテンツ構造**: プロジェクトベースのレイアウトが分かりやすい
2. **視覚的な魅力**: ヒーローセクションの2パネルデザインが印象的
3. **多言語対応**: 一部ページで英語版がある
4. **プロジェクトの明確化**: 「白鷺三十六景」「香港ネオン残照」が際立っている

### ⚠️ 改善が必要な点

#### 1. **パフォーマンス**
- **問題**: 画像の最適化が不十分、フォント読み込みの最適化不足
- **影響**: 初回読み込みが遅い、特にモバイルで体感速度が低下
- **優先度**: 🔴 高

#### 2. **コードの一貫性**
- **問題**: `about.html`がTailwind CSSを使用、他はカスタムCSS
- **影響**: メンテナンス性の低下、スタイルの不統一
- **優先度**: 🟡 中

#### 3. **SEO最適化**
- **問題**: 構造化データ（JSON-LD）がない、OGP画像の最適化不足
- **影響**: 検索エンジンでの表示品質、SNSシェア時の見栄え
- **優先度**: 🟡 中

#### 4. **アクセシビリティ**
- **問題**: ARIAラベルの不足、キーボードナビゲーションの不完全性
- **影響**: スクリーンリーダーユーザーへの対応不足
- **優先度**: 🟡 中

#### 5. **モバイルUX**
- **問題**: 一部のインタラクションがモバイルで最適化されていない
- **影響**: モバイルユーザーの体験低下
- **優先度**: 🟡 中

#### 6. **コードの重複**
- **問題**: ナビゲーション、フッター、メタタグが各ページに重複
- **影響**: メンテナンスコストの増加
- **優先度**: 🟢 低

## 🎯 改善提案（優先順位順）

### 🔴 優先度：高

#### 1. 画像最適化とパフォーマンス改善

**現状の問題**:
- 画像がWebP形式に変換されていない
- `loading="lazy"`は一部のみ
- 画像サイズの最適化が不十分

**改善策**:
```html
<!-- 例: モダンな画像フォーマットの使用 -->
<picture>
  <source srcset="image.webp" type="image/webp">
  <source srcset="image.avif" type="image/avif">
  <img src="image.jpg" alt="..." loading="lazy" decoding="async">
</picture>
```

**実装項目**:
- [ ] 主要画像をWebP/AVIF形式に変換
- [ ] `loading="lazy"`を全画像に適用（ヒーロー画像以外）
- [ ] 画像のサイズ最適化（適切な解像度）
- [ ] `decoding="async"`の追加

**期待効果**: ページ読み込み速度 30-50% 改善

---

#### 2. フォント読み込みの最適化

**現状の問題**:
- Google Fontsが同期読み込み
- フォント表示の遅延（FOUT/FOIT）

**改善策**:
```html
<!-- フォントの事前読み込み -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=..." rel="stylesheet" media="print" onload="this.media='all'">
<noscript><link href="https://fonts.googleapis.com/css2?family=..." rel="stylesheet"></noscript>
```

**実装項目**:
- [ ] `preconnect`の追加
- [ ] フォントの非同期読み込み
- [ ] `font-display: swap`の設定
- [ ] 使用していないフォントウェイトの削除

**期待効果**: フォント読み込み時間 40-60% 短縮

---

### 🟡 優先度：中

#### 3. コードの一貫性向上

**現状の問題**:
- `about.html`のみTailwind CSS使用
- 他のページはカスタムCSS

**改善策（2つの選択肢）**:

**選択肢A: Tailwindを全ページに統一**
- メリット: 開発速度向上、一貫性確保
- デメリット: 既存CSSの書き直しが必要

**選択肢B: カスタムCSSに統一（推奨）**
- メリット: 既存コードの活用、バンドルサイズ削減
- デメリット: カスタムCSSのメンテナンスが必要

**推奨**: 選択肢B（カスタムCSS統一）
- `about.html`のTailwindをカスタムCSSに移行
- 共通コンポーネントのCSSモジュール化

**実装項目**:
- [ ] `about.html`のスタイルを`style.css`に統合
- [ ] 共通コンポーネント用CSSファイルの作成
- [ ] スタイルガイドの作成

---

#### 4. SEO最適化の強化

**現状の問題**:
- 構造化データ（JSON-LD）がない
- OGP画像のサイズ最適化不足
- メタディスクリプションが一部ページで不足

**改善策**:

**構造化データの追加**:
```json
{
  "@context": "https://schema.org",
  "@type": "Person",
  "name": "Toji Masaya",
  "jobTitle": "Photographer, Writer",
  "url": "https://tojimasaya.com",
  "sameAs": [
    "https://instagram.com/tojimasaya",
    "https://note.com/tojimasaya"
  ]
}
```

**実装項目**:
- [ ] Person型の構造化データ追加（Aboutページ）
- [ ] Article型の構造化データ追加（Writingsページ）
- [ ] ImageObject型の構造化データ追加（Galleryページ）
- [ ] OGP画像を1200x630pxに統一
- [ ] 全ページにメタディスクリプション追加

**期待効果**: 検索エンジンでの表示品質向上、リッチスニペット表示

---

#### 5. アクセシビリティの向上

**現状の問題**:
- ARIAラベルの不足
- キーボードナビゲーションの不完全性
- コントラスト比の確認不足

**改善策**:

**ARIAラベルの追加**:
```html
<nav aria-label="メインナビゲーション">
<button aria-label="メニューを開く" aria-expanded="false">
<img alt="白鷺三十六景 — 姫路城" loading="lazy">
```

**実装項目**:
- [ ] 全ナビゲーションに`aria-label`追加
- [ ] モーダルに`aria-modal="true"`追加
- [ ] 画像に適切な`alt`属性確認
- [ ] キーボード操作のテスト（Tab、Enter、Esc）
- [ ] コントラスト比の確認（WCAG AA準拠）

**期待効果**: アクセシビリティスコア向上、より多くのユーザーに対応

---

#### 6. モバイルUXの改善

**現状の問題**:
- 一部のインタラクションがタッチデバイスで最適化不足
- ヒーローセクションの高さがモバイルで大きすぎる可能性

**改善策**:

**タッチターゲットサイズの最適化**:
```css
/* 最小タッチターゲット: 44x44px */
.nav-link, .filter-btn {
  min-height: 44px;
  min-width: 44px;
}
```

**実装項目**:
- [ ] タッチターゲットサイズの確認（最小44x44px）
- [ ] スワイプジェスチャーの追加（ギャラリー）
- [ ] モバイルでのヒーロー高さ調整
- [ ] モバイルメニューのアニメーション改善
